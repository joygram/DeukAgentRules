# 작동 구조

이 저장소는 AI 코딩 에이전트를 위한 워크플로우 제어면을 제공합니다. 단순히 룰 파일만 생성하는 도구가 아니라, 작업 범위 설정, 티켓 발급, 파일 변경 가능 시점을 함께 정의합니다.

## 기본 개념

- `README.md`는 진입점이에요.
- `AGENTS.md`는 정본 실행 규칙 파일이에요.
- 티켓은 하나의 경계가 분명한 작업에 대한 실행 계약서예요.
- 플랜 문서는 그 티켓의 설계 계층이에요.
- 대상 서브모듈은 작업 범위 경계예요.

| 경로 | 역할 |

| `README.md` | 상위 개념 진입점과 탐색 문서 |

| `.deuk-agent/docs/walkthroughs/` | 완료 보고서와 사후 정리 문서 |

## 워크플로우 구조

1. `deuk-agent-rule init`으로 작업 공간을 초기화해요.
2. 범위가 정해진 문서나 코드를 바꾸기 전에 티켓을 발급해요.
3. 설계는 먼저 플랜 문서에 적어서 실행 전에 명시해요.
4. 실행은 선언된 대상 서브모듈 안에서만 진행해요.
5. 완료 전에는 검증과 이슈 기록을 남겨요.
6. 작업이 끝나면 티켓을 아카이브해요.

## 구조가 필요한 이유

- 티켓 범위를 좁혀서 컨텍스트 크기를 줄일 수 있어요.
- 설계와 변경을 분리할 수 있어요.
- 승인 경계를 암묵값이 아니라 명시값으로 만들 수 있어요.
- RAG가 과거 결정을 안정적으로 검색할 수 있어요.

## 문서 경계

이 문서는 구조를 설명해요. `AGENTS.md`의 모든 규칙을 다시 쓰지는 않아요.

다음 내용을 이해하고 싶을 때 이 파일을 보면 돼요.

- 티켓이 실행 범위와 어떻게 연결되는지
- 설계 계층이 어디에 있는지
- 에이전트 워크플로우가 어떻게 단계별로 분리되는지
- README를 왜 짧고 탐색용으로 유지하는지

워크플로우를 이렇게 설계한 이유는 `docs/principles.ko.md`에서 설명해요.
